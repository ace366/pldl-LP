## 目的

PLDL-LP の営業フェーズ用に、学童施設の営業候補リストを100件作成・管理する軽量ツールを作成する。

このツールは、既存の Laravel 製 LP 本体とは完全に分離して作成する。

---

## 現在のプロジェクト状況

作業場所：

```txt
C:\work\PLDL-LP

この直下には、すでに Laravel プロジェクトが存在している。

例：

C:\work\PLDL-LP
├ app
├ bootstrap
├ config
├ database
├ public
├ resources
├ routes
├ storage
├ vendor
├ artisan
├ composer.json
├ package.json

そのため、営業リストツールは Laravel 本体に影響しない形で作成する。

最重要方針

営業リストツールは Laravel とは完全分離する。

以下は触らないこと。

app/
routes/
resources/
database/
config/
bootstrap/
vendor/
artisan
package.json
vite.config.*

Laravel のルート追加、Controller追加、Vite設定変更はしない。

採用する構成

営業リストツールは以下に作成する。

C:\work\PLDL-LP\sales-tool

構成：

sales-tool/
├ index.html
├ assets/
│   ├ sales-tool.css
│   └ sales-tool.js
└ README.md

Laravel本体とは独立した、純粋な HTML + CSS + JavaScript のローカルツールとして作る。

なぜこの構成にするか

Laravel本体に組み込むと、以下のリスクがある。

LP本体の表示に影響する
Vite build が壊れる可能性がある
routes/web.php の競合が起きる
本番デプロイ対象が複雑になる
営業ツール修正でLP本体に事故が起きる

今回の目的は、まず営業リスト100件を作ることなので、Laravelに組み込まない。
