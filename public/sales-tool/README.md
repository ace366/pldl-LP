# PLDL 営業リスト管理ツール

学童施設の営業候補リスト100件を作成・管理する**軽量フロントエンドツール**。

Laravel のルート / Controller / migration / vite には一切依存していません。`public/sales-tool/` 配下の静的ファイルのみで完結します（Web サーバの静的配信に乗っているだけ）。

## 起動方法

### 本番環境

```
https://top-ace-picard.sakura.ne.jp/pldl-lp/sales-tool/
```

Laravel のルーティングを通らず、さくらの静的配信で直接届きます。

### ローカル開発

`php artisan serve` を立ち上げてあれば:

```
http://localhost:8000/sales-tool/
```

または `file://` でも動きます:

```
file:///C:/work/PLDL-LP/public/sales-tool/index.html
```

## データの保存先

ブラウザの **localStorage** に保存（キー: `pldl_sales_list_v1`）。

- ブラウザを閉じても消えません
- ただし**ブラウザごと・端末ごと**の保存です
- 別端末で使う場合は **JSON書き出し → 別端末で取り込み** で同期

## 機能

| カテゴリ | 機能 |
|---|---|
| 入力 | 施設名 / 都道府県 / 市区町村 / 住所 / 電話 / Webサイト / 問い合わせフォーム / GoogleMap / 種別 / 優先度 / ステータス / メモ / 初回送信日 / 次回対応日 |
| 種別 | 民間学童 / 放課後児童クラブ / NPO / 法人運営 / 公立・自治体系 / その他 |
| 優先度 | S / A / B / C（自動判定ボタン付き） |
| ステータス | 未送信 / 送信済み / 返信あり / 商談 / 契約 / 見送り / 対象外 |
| 一覧 | KPI 5枚（総数・進捗バー・S件数・未送信・返信あり・契約）/ 検索 / 優先度・ステータス絞り込み |
| 表示 | PC: テーブル / スマホ: カード（自動切替） |
| 編集 | 一覧の「編集」または「営業文」ボタン |
| バックアップ | JSON書き出し / JSON取り込み（上書き or 追加を選択） / CSV書き出し（Excel対応BOM付） |
| 営業文 | 「営業文」ボタンで施設名入りテンプレをクリップボードにコピー |

## 100件作成の使い方

1日10件目安で進める。

1. **+ 新規追加** で施設を入力
2. **自動判定** ボタンで優先度の候補を入れる（種別・URL有無から推定）
3. ステータスは初期 `未送信` のまま保存
4. 1日終わったら **JSON書き出し** でバックアップ

### 1日10件の内訳目安
- Googleマップから5件
- 自治体サイト一覧から3件
- 民間運営会社から2件

### 100件目標の優先度内訳
- S: 20件 / A: 30件 / B: 30件 / C: 20件

## JSON 書き出し / 取り込み

- **書き出し**: ヘッダー右の「JSON 書き出し」ボタン → `pldl-sales-list-YYYY-MM-DD.json` がダウンロード
- **取り込み**: 「JSON 取り込み」ボタン → ファイル選択 → 「上書き / 追加」を選ぶ

## CSV 書き出し

- ヘッダー右の「CSV 書き出し」ボタン → `pldl-sales-list-YYYY-MM-DD.csv`
- BOM付き UTF-8 なので Excel でそのまま開ける

## 営業文テンプレ

「営業文」ボタンで施設名差し込み済みの本文がクリップボードにコピーされる。テンプレ内容は `sales-tool.js` の `SALES_TEMPLATE` 定数で編集可能。

## Laravel 本体への影響

なし。`public/sales-tool/` を丸ごと削除しても LP / 管理画面 / パンフレットは動作します。`public/` 配下の静的ファイルとして Web サーバが返しているだけで、Laravel のルート (`routes/web.php`) や Controller には何も追加していません。

## 公開範囲の注意

`public/sales-tool/` は **誰でも URL を知っていれば開けます**（認証なし）。データはブラウザの localStorage に閉じているので他人のリストは見えませんが、ツール自体は公開状態です。検索エンジン避けは `<meta name="robots" content="noindex,nofollow">` で対応済み。

社外公開したくなければ `.htaccess` で IP / Basic 認証を追加するか、最初の構成（`file://` でローカル開く）に戻してください。
